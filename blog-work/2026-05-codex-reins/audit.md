# 为 Codex 套上缰绳：审计记录

核心主张是，Codex 的主要风险不是回答错误，而是带副作用的执行缺少边界。根据用户反馈，文章从偏工程说明的版本回调为更接近原始播客草稿的叙述方式：从社交媒体误操作案例进入，用“野马”和“缰绳”贯穿全文，最后把可复制的全局 `AGENTS.md` 规则作为 `text` 代码块放在文末。

证据来源包括用户提供的原始观点和规则清单、当前站点模板 `templates/content-template.md`、现有博客 `src/content/blog/2026-01-skill-guardrails-audit.md` 的 frontmatter 与中文长文风格，以及 OpenAI 官方 Codex 文档中的 AGENTS.md 说明。官方文档支持文章里关于全局 `~/.codex/AGENTS.md`、项目级 `AGENTS.md`、层级合并、靠近当前目录的规则覆盖更宽泛规则、默认大小限制和会话启动时读取指令链的说法。文档链接是 https://developers.openai.com/codex/guides/agents-md 。

未验证主张是，社交媒体上关于 agent 误删文件、破坏系统路径或污染环境的案例没有逐条检索和引用。文章保留了这一类现象作为用户给定背景，但没有命名具体事件、平台、账号或可归责产品，因此没有把未经核验的轶事写成事实证据。

本次修订移除了上一版过多的分节和偏工程规范语气，保留规则解释，但让叙述更接近用户原始表达。文章没有展开“AI 安全很重要”这类宽泛开场，没有使用“随着技术发展”“本文将”“总而言之”等模板式转场。正文仍覆盖删除、git 状态、未提交改动、最小 diff、依赖作用域、命令说明、敏感信息、设计源、项目规则和最终交付证据。

上一版质量门前几次运行时提示正文低于默认中文长文长度，因此扩展了全局规则与项目规则的分工、发布门、本地状态与远端状态的差异、权限和信任的关系、失败动作，以及 `tmp/archive_YYYYMMDD_HHMMSS/` 作为恢复缓冲区和审计缓冲区的机制。本次回调先缩短了部分工程展开并新增文末原样规则块，随后根据质量门提示补充了更接近原始叙述风格的连续段落，重点讲清 Codex 连续动作、prompt 临时提醒的不足、全局规则和项目规则的分工、状态污染、底线规则，以及规则如何成为工作流记忆。扩展没有新增未经核验的外部事故案例。

需要作者后续确认的问题是，是否希望未来补充真实公开事故案例作为开场证据，是否要把文中的全局 AGENTS.md 协议整理成可复制模板，以及是否要为不同项目类型分别给出小说、科研代码、静态网站和部署仓库的项目级 AGENTS.md 示例。
